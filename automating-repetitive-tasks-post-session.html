Level 1 — Reduce Thinking
Format Transformer

This prompt reformats a source CSV into Asana-ready structure — the same transformation rules, applied reliably every time. Paste your CSV content and reference document into a chatbot, run this prompt, and import the output directly.

The Prompt
You are a helpful assistant helping to organize data in a csv.
You will be given a CSV with the following relevant information:
  column 3: QID
  column 4: Categories
  column 5: Subcategory
  column 6: Topic

You should create a new CSV containing the following information:
  column 1: Task (QID from column 3)
  column 2: Specialty (Primary)
  column 3: Project Title(s)
  column 4: Section
  column 5: Category
  column 6: Subcategory
  column 7: QID Topic (Topic from column 6)
  column 8: QID Questions (always 1)

Specialty (Primary) = "Emergency Medicine"
Project Title(s) = "ECG Fundamental and Advanced Qbank Review"
Section = "Triage (Review Content)"

The fields for Categories and Subcategories both contain multiple
values, separated by commas. To identify which of the values to
bring into the new spreadsheet, you must search the attached
Reference document. For Category, you will match one of the values
in column 4 of the original sheet to column 1 of the reference
sheet and report back the matching value. For Subcategory, you will
match one of the values in column 5 of the original sheet to column
2 of the reference sheet and report back the matching value.

Level 2 — Reduce Steps
Classifier: callGPT Function

Paste this function into Google Apps Script (Extensions → Apps Script). Once it’s in, use =callGPT(systemMessage, userPrompt) as a formula directly in any cell. Start at Level 1 first — only move here once your prompt is producing reliable output.

Google Apps Script
function callGPT(systemMessage, userPrompt) {
  var apiKey = 'Your-API-Key-Here'; // Replace with your real API key
  var url = 'https://api.openai.com/v1/chat/completions';
  var payload = {
    model: "gpt-4", // Or "gpt-3.5-turbo" for lower cost
    messages: [
      { role: "system", content: systemMessage },
      { role: "user",   content: userPrompt }
    ],
  };
  var options = {
    method: 'post',
    contentType: 'application/json',
    headers: { Authorization: 'Bearer ' + apiKey },
    payload: JSON.stringify(payload)
  };
  try {
    var response = UrlFetchApp.fetch(url, options);
    var json = JSON.parse(response.getContentText());
    return json.choices[0].message.content.trim();
  } catch (error) {
    return "Error: " + error.toString();
  }
}

Level 3 — Remove Yourself
Trigger → Action → Output

This is the Zapier workflow that pulls a Looker report monthly, classifies each entry, and writes the results directly into a Google Sheet. You’re only in the loop at the end — reviewing output, not producing it. Build this in Zapier, n8n, or Make.

Workflow Spec (Zapier / n8n / Make)
1. Pull the Looker report for the previous month.
   [Looker: Run Look]

2. Identify the Month for which the report is for.

3. Analyze each entry to determine if the work done was for
   Content (reviewing questions, writing questions, etc)
   or Production (storyboarding).

4. Create as many rows as needed in the Google Sheet so that
   each entry gets one line. Each row should contain:
     • Payroll Category
     • First Name
     • Last Name
     • Month
     • Duration (formatted as a number with no text)
     • Notes
     • Whether the work was for Content or Production

   [Google Sheets: Create Spreadsheet Row]
Before you trust it to run solo, check:
Reflect

What’s a monthly or weekly output you produce that has a predictable trigger and a consistent destination? Can you fill in: “When [X happens], I want [Y to happen], and the result should go to [Z]”?
