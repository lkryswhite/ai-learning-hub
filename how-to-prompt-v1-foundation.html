The reframe
The core idea
Most people treat it like a search engine.
It's not. It's a contractor you're briefing.
Search engine
Retrieves what already exists. Vague input is fine — it filters for you.
Contractor
Executes what you spec. Vague input produces average output. The brief is everything.
Anatomy of a good prompt
ComponentWhat it doesExample
Role / Audience Sets the lens — who's writing AND who's reading. Audience spec does more work than role alone. "Write for a pre-med junior two weeks from their exam who finds biochemistry intimidating. Plain language, no assumed knowledge."
Context Gives background it doesn't have. Lead with task, support with context. "We're building a study guide for students 2 weeks from their exam who are overwhelmed and skimming."
Task Tells it exactly what to do. Vague tasks get vague outputs. "Write three explanations of cellular respiration at increasing difficulty levels."
Format Defines what the output looks like — length, structure, tone. "Bulleted list, under 100 words each, plain language, no jargon."
The Golden Rule Test: Could a smart colleague with no background follow it without being confused? If they'd be confused, the model will be too. That check catches most bad prompts before they happen.
When output misses: three moves
1
Tell it what's wrong — specifically
Name the actual problem. Vague feedback produces vague revision.
"This is too formal — rewrite it the way you'd explain it to a colleague over Slack."
2
Add a constraint
Give it a hard rule to work within.
"Keep this under 50 words." / "No bullet points — write it as a single paragraph."
3
Show it what good looks like
Paste an example. The model mimics what it sees more reliably than instructions.
"Here's the style I'm going for: [paste]. Write mine the same way but for [your use case]."
Key nuance — replace "don't" with "do"
"Don't be verbose."
→
"Write concisely — under 150 words."
Negation fails reliably. Positive constraints almost always work better.
The iteration template
Steal this
Here's what I got: [paste output]. This is close but [specific problem — too formal / too long / missing X]. Can you revise it to [specific fix]? Keep [what to preserve].
Stay in the same conversation. Expert users expect 3–8 turns of refinement on any nontrivial task. The accumulated context is worth more than anything you'd type fresh. When threads get very long and quality degrades: ask it to summarize → open a new chat → paste the summary.
Do this today
Try It
Take one prompt you've used in the last week. Rebuild it using the four-component anatomy.
  • 1Write out who's actually reading the output — specifically, not just "a student" or "my manager."
  • 2Check your original prompt against Role/Audience, Context, Task, Format. Which one is missing?
  • 3Add it. Run the prompt. Compare what you get now to what you got before.
  • 4If the first output still misses, use one of the three iteration moves — don't rewrite from scratch.
Reflect
Think of a task you've tried with AI that didn't work well. Which of the four components — Role/Audience, Context, Task, or Format — was weakest in your prompt? What would you add now?
